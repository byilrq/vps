# Claude 交互规则 – 极致降低 Token 消耗

## 核心原则
- **直接修改文件**：所有代码变更必须直接编辑目标文件，禁止输出补丁、diff 或“请手动修改”。
- **完全禁用本地校验**：不运行任何测试、编译、构建、安装命令，所有验证由用户手动执行。
- **不安装任何工具**：不推荐安装软件包或依赖，只提供查看类 Linux 命令（如 `cat`, `grep`, `tail`, `ss`, `ps`, `journalctl`）。
- **问题诊断**：仅提供一条排查命令，用户贴出完整输出后，我基于输出只给一次修改，不再追加询问。

## 具体行为规范

### 1. 文件操作
- 使用 `write_to_file` 或 `replace_in_file` 直接写入或替换文件内容。
- 不添加注释、调试信息或无关改动。
- 若需删除文件，使用 `delete_files` 并附一行原因。

### 2. 代码修改
- 默认无注释，除非原文件有类似风格。
- 最小改动原则，不重构无关部分。
- 不确定时先提问（仅一次），不猜测。

### 3. 错误处理
- 不输出推理过程，只给出直接修改或单条命令。
- 不重复用户提供的错误信息。
- 仅提供静态查看命令（如 `cat`, `tail`, `grep`, `systemctl status` 等），不提供安装、编译、重启（除非明显安全且必要，但重启命令也不建议自动执行）。

### 4. 对话效率
- 回答不超过 3 句话（修改文件时，只说明改了哪个文件；提供命令时，只给命令）。
- 不提供多种方案，只给一种。
- 不主动询问偏好，除非必须二选一（例如“用端口 8080 还是 8081？”），且只问一次。

### 5. 例外
- 若您明确要求解释、提供多种方案或运行某命令，可破例。

## 示例交互
**您**：Nginx 502，日志显示 connection refused。  
**我**：已修改 upstream 地址为 127.0.0.1:3000。请执行 `sudo nginx -t` 并贴出结果。

**您**：提示 unknown directive。  
**我**：已删除该 directive。请再次执行 `sudo nginx -t`。
